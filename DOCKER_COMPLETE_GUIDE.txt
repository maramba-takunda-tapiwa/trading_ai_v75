================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Dashboard blank/white screen
SOLUTION:
- Wait 10 seconds for services to start
- Refresh browser (Ctrl+R or Cmd+R)
- Check: docker ps (should show trading-ai-dashboard)
- If not running: docker-compose up -d

PROBLEM: "Connection refused" error
SOLUTION:
- Container not running: docker-compose up -d
- Wait 5 seconds for services to start
- Try: http://localhost/api/health
- View logs: docker-compose logs trading-ai

PROBLEM: No trades appearing
SOLUTION:
- Normal if just deployed
- EUR/USD only trades during breakouts
- Wait 1+ hours for first trades
- Check raw data: http://localhost/api/trades

PROBLEM: Port 80 already in use
SOLUTION:
- Edit docker-compose.yml
- Change: "80:80" to "8080:80"
- Save and run: docker-compose up -d
- Access: http://localhost:8080

PROBLEM: Container crashes on startup
SOLUTION:
- Check logs: docker-compose logs trading-ai
- Rebuild: docker-compose build --no-cache
- Restart: docker-compose down; docker-compose up -d
- Check disk space: Need ~2GB

PROBLEM: Data not persisting
SOLUTION:
- Data is in ./results/ folder (persists)
- Also in Docker volume 'trading-ai-data'
- Check: ls results/
- Should show: live_trades_log_v2.csv, etc.
